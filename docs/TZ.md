# ТЗ: amoCRM Analytics Appliance

## Цель

Сделать отдельный self-hosted продукт для анализа продаж по данным amoCRM: подключение, импорт, конструктор отчетов, рабочий стол РОПа.

## Роли

- `ADMIN`: подключает amoCRM, управляет пользователями, видимостью менеджеров/групп, настройками прогноза и дашборда.
- `ROP`: смотрит рабочий стол, собирает отчеты, экспортирует Excel, видит только разрешенных менеджеров и группы.

## Синхронизация

- Full sync: вся доступная история.
- Incremental sync: периодическая догрузка по `updated_at` с защитным overlap.
- Webhook sync: быстрый триггер на изменения amoCRM.
- Все операции идемпотентны, повторы не создают дублей.

## Конструктор отчетов

Поддерживаются два источника данных:

- `event`: переходы между этапами, вход в этап, выход из этапа, создание, закрытие, смена ответственного, изменение полей.
- `current`: текущий этап, текущая сумма, текущий ответственный, текущая воронка, текущие значения полей.

Фильтры: период, воронка, этапы, менеджеры, группы amoCRM, кастомные поля, теги, причины отказа, сумма, даты, статус.

## Ключевые отчеты

- Конверсии: ordered steps, denominator `previous` или `first`, разрез по отделу, менеджеру, группе.
- Forecast: выбранный закрывающий этап + отгрузочная воронка + weighted pipeline.
- Плечо отгрузки: среднее и медианное время от входа в отгрузочную воронку до успешного этапа.

## Приемка

- Нет hardcoded amoCRM IDs.
- Отчеты по переходам считаются через историю, не через текущий snapshot.
- `fromStageId === toStageId` не считается бизнес-переходом.
- При малой выборке auto probability показывает низкую confidence, а не выдуманную точность.
- Excel совпадает с экранным отчетом по фильтрам.
