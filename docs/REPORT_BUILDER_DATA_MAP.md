# Карта данных визуального конструктора отчётов

## Цель

Сделать конструктор, в котором пользователь собирает отчёт кликами:

- строки: менеджеры или группы менеджеров;
- столбцы: показатели;
- ячейки: значение показателя для строки;
- итоги: сумма, среднее, минимум, максимум, процент;
- формулы: расчёт из других показателей.

## Структура отчёта

### Основные поля

- название отчёта;
- описание отчёта;
- кому виден отчёт;
- строки: менеджеры или группы;
- столбцы: показатели;
- итоги по строкам;
- итоги по столбцам.

### Столбец-показатель

У каждого столбца должно быть:

- название;
- описание;
- тип результата: число, деньги, процент, дни, часы;
- сущность: сделки, задачи, примечания, события, товары;
- действие или состояние;
- фильтры;
- период;
- способ расчёта.

### Формульный столбец

Формула должна уметь ссылаться на другие показатели.

Пример:

```txt
Конверсия = Успешные сделки / Все сделки * 100
```

В формулах нужны:

- сложение;
- вычитание;
- умножение;
- деление;
- скобки;
- проценты;
- округление;
- защита от деления на ноль.

## Важное ограничение

Сейчас мы сохраняем не все поля amoCRM отдельными колонками. Часть данных лежит в raw JSON.

Это нормально для старта, но для быстрых отчётов основные поля лучше вынести в отдельные колонки позже.

### Забираем частично

- теги: сохраняем названия тегов внутри сделки, но не храним отдельный справочник тегов;
- товары: сохраняем товары внутри сделки, но не синхронизируем полный каталог товаров;
- связи сущностей: часть связей берём из сделки, но не синхронизируем отдельную таблицу всех связей;
- события: сохраняем события, но пока не превращаем все типы событий в удобные аналитические поля;
- raw-данные: сохраняем, но не все поля вынесены в отдельные колонки для быстрых отчётов.

## Вывод по полноте данных

Текущий слепок подходит для первой версии аналитики по сделкам.

Но для конструктора уровня "накликать почти любой отчёт по amoCRM" нужно расширить синхронизацию.

Минимальный правильный путь:

1. Сначала расширить слепок данных.
2. Потом строить универсальный конструктор.
3. Не пытаться делать полный конструктор поверх неполного слепка.
